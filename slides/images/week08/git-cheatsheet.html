Git Cheatsheet

Local Basics — What happens before & after each command
Working Directory
Your actual files on disk
git add →
Staging Area
What will go in next commit
git commit →
Repository (.git)
Permanent history
Setup
git init Create a new repository
Before $ ls -a
. .. train.py data/
After $ ls -a
. .. .git/ train.py data/
Tip: The .git/ folder stores all history. Delete it = no more repo.
git config Set your identity (required before first commit)
$ git config --global user.name "Your Name"
$ git config --global user.email "you@example.com"
Key Flags
--global Set for all repos on this machine
--local Set for this repo only (default)
--list Show all current settings
Tracking Changes
git status See what's changed
output On branch main

Changes not staged for commit:
  modified: train.py

Untracked files:
  utils.py

Changes to be committed:
  new file: config.yaml
Key Flags
-s / --short Compact output: M modified, ?? untracked, A staged
--short A config.yaml
M train.py
?? utils.py
git add Stage changes for next commit
$ git add train.py utils.py
Before Staging area: config.yaml
Unstaged: train.py
Untracked: utils.py
After Staging area: config.yaml

              utils.py

Key Flags
git add . Stage everything in current directory
git add -p Interactive: pick individual hunks to stage
git add *.py Stage all Python files (glob pattern)
git commit Save a snapshot to history
$ git commit -m "Add training script and utilities"
Before Staged: config.yaml, train.py, utils.py
Log:
a1b2c3d Initial commit
After Staged: (empty — clean!)
Log:
f4e5d6c Add training script and utilities
a1b2c3d Initial commit
Key Flags
-m "msg" Commit message inline (skip editor)
-am "msg" Add all tracked modified files + commit (skip git add)
--amend Replace last commit (fix message or add forgotten files)
git diff Show exactly what changed

What Each Variant Shows
git diff Working dir vs staging (unstaged changes)
git diff --staged Staging vs last commit (what will be committed)
git diff HEAD Working dir vs last commit (all changes)
git diff abc..def Between two commits
--stat Summary: files changed, insertions, deletions
--stat train.py | 3 ++-
utils.py | 12 ++++++++++++
2 files changed, 14 insertions(+), 1 deletion(-)

Variants
git show HEAD Latest commit
git show HEAD~2 Two commits ago
git show :file.py Staged version of a file
Branching
git branch List, create, or delete branches
$ git branch feature/augment
Before Branches:
* main
After Branches:
* main
  feature/augment
(still on main — just created the branch)
Key Flags
git branch List all local branches
git branch -d name Delete branch (safe: only if merged)
git branch -v Show last commit on each branch
git switch Move to a different branch
$ git switch feature/augment
Before HEAD → main
Files: train.py (main version)
After HEAD → feature/augment
Files: train.py (branch version)
Working dir files change!
Key Flags
git switch -c name Create + switch in one step
git checkout name Older syntax (still works, same thing)
Key insight: Switching branches changes your actual files on disk. Git swaps them instantly.
git merge Combine branches together
$ git switch main
$ git merge feature/augment
Before * c3 (feature/augment)
* c2
* c1 (HEAD -> main)
After (fast-forward) * c3 (HEAD -> main, feature/augment)
* c2
* c1
main pointer just moved forward
When both branches have new commits → three-way merge:
Before * c4 (feature)
| * c3 (main)
|/
* c2
After (merge commit) *   c5 (main) Merge feature
|\
| * c4 (feature)
* | c3
|/
* c2
Merge Conflicts When Git can't auto-merge
Both branches changed the same line → Git marks the conflict:
in file <<<<<<< HEAD
lr = 0.001
=======
lr = 0.0005
>>>>>>> feature/augment
To resolve:
# 1. Edit the file — keep what you want, delete markers
$ git add train.py # 2. Stage the resolved file
$ git commit # 3. Finish the merge
Undoing Things
git restore Discard changes or unstage files
$ git restore train.py # discard working dir changes
$ git restore --staged train.py # unstage (keep changes)
Before restore Working dir: train.py (modified)
Staging: (empty)
After restore Working dir: train.py (back to last commit)
Staging: (empty)
Before --staged Working dir: train.py (modified)
Staging: train.py
After --staged Working dir: train.py (still modified)
Staging: (empty — unstaged)
git reset Move HEAD back (undo commits)
$ git reset --soft HEAD~1 # undo last commit, keep staged
$ git reset HEAD~1 # undo commit + unstage (default: --mixed)
$ git reset --hard HEAD~1 # undo everything (DESTRUCTIVE)
--soft
Keeps staging
Keeps working dir
--mixed
Clears staging
Keeps working dir
--hard
Clears staging
Clears working dir
Warning: --hard permanently destroys uncommitted changes!
git stash Temporarily shelve changes
$ git stash # save changes, clean working dir
$ git stash pop # restore changes, remove from stash
Before stash Working dir: train.py (modified)
Stash: (empty)
After stash Working dir: clean!
Stash: stash@{0}: WIP on main
Key Flags
git stash list Show all stashes
git stash pop Apply + delete top stash
git stash apply Apply but keep in stash
Use case: Need to switch branches but have uncommitted work? Stash it, switch, do your thing, switch back, pop.
git revert Undo a commit safely (creates new commit)
$ git revert f4e5d6c
Before * f4e5d6c Add training script
* a1b2c3d Initial commit
After * g7h8i9j Revert "Add training script"
* f4e5d6c Add training script
* a1b2c3d Initial commit
reset vs revert: reset rewrites history (use locally). revert adds a new undo commit (safe for shared branches).
.gitignore
.gitignore Tell Git which files to skip
.gitignore # Python
__pycache__/
*.pyc
.env

# ML artifacts
*.pkl
*.h5
models/
data/raw/

# IDE
.vscode/
.idea/

# OS
.DS_Store
Thumbs.db
Before .gitignore $ git status
?? __pycache__/
?? .env
?? model.pkl
?? train.py
After .gitignore $ git status
?? .gitignore
?? train.py
(junk files hidden!)
Tip: github.com/github/gitignore has templates for every language. Start with Python + your IDE.
